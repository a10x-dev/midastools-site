# Declare a funnel dead with de-confounded full-window data

Purpose: Make a defensible kill call on a funnel by reading every surface separately and ruling out deliverability/measurement before concluding "audience won't convert"

When to use: ~0 across several sessions and you're tempted to declare it dead OR to keep adding traffic/bridges. Before either, run the decisive read.

---

When to use: A funnel has read ~0 across several sessions and you're tempted to declare it dead OR to keep adding traffic/bridges. Before either, run the decisive read.
---
1. Pull the FULL post-event window (not a truncated slice) — the zero must span the whole window, not a 3-hour artifact.
2. Read each funnel surface SEPARATELY: email→tool, on-site peak-intent CTA, blog bridges. A surface-by-surface zero is far stronger than an aggregate zero.
3. Code-verify the highest-intent surface's CTA is structurally sound (single form/anchor, tracked navigation) — so the zero is behavior, not a broken funnel.
4. De-confound deliverability: if email is a surface, get the delivery read (Resend export / dashboard). If delivery is healthy and on-site surfaces ALSO read 0, the verdict is de-confounded — it's real audience behavior, not spam-foldering.
5. Only after 2-4 hold: declare the funnel dead. Reframe as product-SHAPE mismatch (what WILL they pay for?) not audience-mismatch (do they exist?). The distinction changes the next move from "find new audience" to "change the offer for the audience you have."
6. Do NOT add more bridges/traffic after a de-confounded zero — that's the discovery-loop trap that burns weeks.
