## Set Your Pricing Context

Paste this at the start of every AI session before using any prompt below:

```
You are a SaaS pricing and positioning strategist. You understand value-based pricing, packaging strategy, and conversion psychology.

Product: [PRODUCT NAME]
Category: [PRODUCT CATEGORY]
What it does: [ONE-SENTENCE DESCRIPTION]
Target user: [PRIMARY PERSONA]
Pricing model: [PER SEAT / FLAT RATE / USAGE-BASED / TIERED / FREEMIUM]

Current plans:
- Free: [WHAT'S INCLUDED, OR "NO FREE PLAN"]
- [PLAN 1 NAME]: $[PRICE]/mo — [KEY FEATURES]
- [PLAN 2 NAME]: $[PRICE]/mo — [KEY FEATURES]
- [PLAN 3 NAME]: $[PRICE]/mo or custom — [KEY FEATURES]

Key competitors and their pricing:
- [COMPETITOR 1]: [THEIR PRICING]
- [COMPETITOR 2]: [THEIR PRICING]

Brand voice: [TONE]
Website: [URL]
```

## Pricing Page Copy Generator

### Full Pricing Page Copy

```
Write complete pricing page copy for [PRODUCT NAME].

Plans:
- [PLAN 1 NAME]: $[PRICE]/mo — Target user: [WHO], Key features: [LIST 5-7]
- [PLAN 2 NAME]: $[PRICE]/mo — Target user: [WHO], Key features: [LIST 5-7]
- [PLAN 3 NAME]: $[PRICE]/mo — Target user: [WHO], Key features: [LIST 5-7]

Annual discount: [PERCENTAGE OR "NONE"]
Free trial: [YES/NO — DURATION]
Money-back guarantee: [YES/NO — DURATION]

Write:

1. Page headline: 3 options (value-focused, not "Pricing")
   - Example directions: "Simple pricing that scales with you" or "Start free, upgrade when you're ready"

2. Page subheadline: One sentence that addresses the #1 pricing concern for your audience

3. For each plan:
   - Plan name
   - Price display (monthly, with annual toggle option)
   - One-line description: Who this plan is for
   - Feature list: [5-7 FEATURES] — lead with the most compelling, not alphabetical
   - CTA button text (different per plan: "Start Free", "Start Trial", "Contact Sales")
   - Badge: Highlight the recommended plan with "Most Popular" or "Best Value"

4. Feature comparison footnote: "All plans include: [LIST 3-5 FEATURES COMMON TO ALL PLANS]"

5. FAQ section: 6-8 pricing-specific questions:
   - Can I switch plans anytime?
   - What happens when my trial ends?
   - Do you offer refunds?
   - Is there a setup fee?
   - What payment methods do you accept?
   - Do you offer discounts for [STARTUPS / NONPROFITS / EDUCATION]?
   - [PRODUCT-SPECIFIC PRICING QUESTION]
   - [PRODUCT-SPECIFIC PRICING QUESTION]

6. Final CTA section: Below the FAQ, one more push to action

Requirements:
- Feature lists should lead with benefits, not feature names ("Unlimited projects" > "Projects: Unlimited")
- The recommended plan should be visually and verbally emphasized
- Pricing should feel simple — if someone needs a calculator to understand it, simplify
```

> **Why this works:** The best pricing pages do three things: (1) help users self-select the right plan instantly, (2) make the recommended plan obvious, and (3) remove purchase anxiety with FAQs and guarantees. Every extra second spent confused is a conversion lost.

### Enterprise / Custom Pricing Section

```
Write enterprise pricing section copy for [PRODUCT NAME].

Context:
- Minimum seat count: [X SEATS OR "FLEXIBLE"]
- Custom features available: [LIST 3-5 — SSO, SAML, custom integrations, SLA, dedicated support]
- Sales process: [BOOK A DEMO / CONTACT SALES / CUSTOM QUOTE]
- Typical enterprise customer: [COMPANY SIZE, INDUSTRY, USE CASE]

Write:
1. Section headline: 3 options (not "Enterprise" — that's boring)
2. One-sentence description of who this is for
3. Feature list: [5-8 ENTERPRISE FEATURES] — each with a brief benefit explanation
4. Social proof: Space for enterprise customer logos + a quote
5. CTA: "Talk to Sales" or "Book a Demo" — plus what to expect:
   "Schedule a 30-minute call. We'll discuss your team's needs and build a custom plan."
6. What happens after they click: Set expectations (timeline, process, no pressure)

Requirements:
- Enterprise copy should feel premium but not intimidating
- Emphasize security, compliance, and support
- Make the sales process feel consultative, not pushy
- Include a "minimum" indication so small teams don't waste their time
```

### Startup / Special Pricing Program

```
Write copy for a startup or special discount program for [PRODUCT NAME].

Program details:
- Who qualifies: [CRITERIA — e.g., "under $1M ARR", "pre-Series A", "nonprofit", "education"]
- Discount: [PERCENTAGE OFF OR SPECIAL PRICING]
- Duration: [HOW LONG THE DISCOUNT LASTS]
- How to apply: [APPLICATION PROCESS]
- What's included: [WHICH PLAN AT WHAT DISCOUNT]

Write:
1. Page headline: 3 options (supportive, community, mission-driven)
2. Subheadline: Who qualifies in one sentence
3. What you get: Plan details with discounted pricing
4. Eligibility criteria: Clear bulleted list
5. Application section: What they need to submit
6. CTA: "Apply for the [PROGRAM NAME]"
7. FAQ: 4-5 questions about the program

Requirements:
- Tone: Genuinely supportive, not performative
- Make the application process simple (if it takes more than 5 minutes, people won't apply)
- Set clear expectations on approval timeline
```

## Feature Comparison Table Builder

### Internal Feature Comparison (Plans)

```
Build a feature comparison table for [PRODUCT NAME]'s pricing plans.

Plans: [PLAN 1], [PLAN 2], [PLAN 3]

Feature categories and features to include:

Category 1: [CORE FEATURES]
- [Feature 1]
- [Feature 2]
- [Feature 3]
- [Feature 4]

Category 2: [COLLABORATION]
- [Feature 5]
- [Feature 6]
- [Feature 7]

Category 3: [INTEGRATIONS & DATA]
- [Feature 8]
- [Feature 9]
- [Feature 10]

Category 4: [SUPPORT & SECURITY]
- [Feature 11]
- [Feature 12]
- [Feature 13]

For each feature, indicate availability per plan:
- Use checkmarks, "X", or specific limits (e.g., "Up to 5", "Unlimited")
- Add a one-line tooltip description for features that need explanation

Requirements:
- Group features logically by category (not alphabetically)
- Place the most differentiating features at the top of each category
- The table should make the value gap between plans clear
- Include a CTA row at the bottom of each plan column
- Recommend which features to REMOVE from the table (too many rows overwhelm — aim for 12-18 total)
```

### External Comparison Table (vs. Competitors)

```
Build a comparison table: [PRODUCT NAME] vs. [COMPETITOR 1] vs. [COMPETITOR 2].

Features to compare:
1. [FEATURE/CAPABILITY 1]
2. [FEATURE/CAPABILITY 2]
3. [FEATURE/CAPABILITY 3]
4. [FEATURE/CAPABILITY 4]
5. [FEATURE/CAPABILITY 5]
6. [FEATURE/CAPABILITY 6]
7. [FEATURE/CAPABILITY 7]
8. [FEATURE/CAPABILITY 8]
9. Pricing (starting price)
10. Free plan/trial
11. [UNIQUE FEATURE YOU HAVE]
12. [UNIQUE FEATURE YOU HAVE]

For each row:
- Rate each product: Yes / No / Partial / Limited / Superior
- Add a brief note where your product is superior
- Be honest where competitors are equal or better

Also write:
- Table headline: 3 options
- Introductory sentence (position the comparison as objective and helpful)
- Footnote about when the comparison was last updated: "[MONTH YEAR]"
- Disclaimer: "We strive for accuracy. If you spot an error, email us at [EMAIL]"

Requirements:
- Include features where you DON'T win (builds credibility)
- Never misrepresent competitor capabilities
- Choose features that highlight your genuine advantages
- Keep the table to 10-15 rows maximum
```

> **Pro tip:** Competitor comparison tables that are 100% green checkmarks for you and 100% red X's for competitors destroy credibility. Include 2-3 rows where competitors match or beat you. Counter-intuitively, this makes the rows where you win MORE convincing.

## Value Proposition Generator

### Core Value Proposition

```
Generate value propositions for [PRODUCT NAME].

Product: [WHAT IT DOES IN DETAIL]
Target user: [PRIMARY PERSONA — be specific about their role, company, and daily challenges]
Key outcomes: [WHAT USERS ACHIEVE — time saved, revenue gained, errors reduced, etc.]
Competitive alternatives: [WHAT THEY USE TODAY — competitors, spreadsheets, manual processes]
Unique mechanism: [HOW YOUR PRODUCT DELIVERS THE OUTCOME DIFFERENTLY]

Generate 5 value proposition frameworks:

1. Before/After:
   "Before [PRODUCT NAME]: [PAIN STATE]. After [PRODUCT NAME]: [DESIRED STATE]."

2. For/Who/That:
   "For [TARGET USER] who [PAIN POINT], [PRODUCT NAME] is the [CATEGORY] that [KEY BENEFIT]. Unlike [ALTERNATIVE], we [DIFFERENTIATOR]."

3. Outcome-First:
   "[SPECIFIC OUTCOME] without [SPECIFIC PAIN]. [PRODUCT NAME] [HOW]."

4. Question Hook:
   "What if you could [DESIRED OUTCOME] in [TIMEFRAME]? [PRODUCT NAME] makes it possible by [HOW]."

5. Social Proof:
   "[NUMBER] [USERS/COMPANIES] use [PRODUCT NAME] to [OUTCOME]. [SPECIFIC METRIC OR RESULT]."

For each framework:
- Write 2 variations
- Explain when to use it (landing page, ads, sales calls, email, etc.)
- Rate its effectiveness for different awareness levels (unaware, problem-aware, solution-aware)
```

### Persona-Specific Value Propositions

```
Write targeted value propositions for [PRODUCT NAME] for different buyer personas.

Personas:

1. [PERSONA 1 — e.g., "Marketing Manager at a mid-market B2B company"]
   - Their pain: [SPECIFIC PAIN]
   - What they care about: [METRICS THEY'RE JUDGED ON]
   - Language they use: [TERMS AND JARGON IN THEIR WORLD]

2. [PERSONA 2 — e.g., "CTO at a startup"]
   - Their pain: [SPECIFIC PAIN]
   - What they care about: [METRICS]
   - Language: [TERMS]

3. [PERSONA 3 — e.g., "VP of Sales at an enterprise company"]
   - Their pain: [SPECIFIC PAIN]
   - What they care about: [METRICS]
   - Language: [TERMS]

For each persona write:
- Headline value prop (8-12 words)
- Subheadline (15-25 words)
- 3 supporting bullet points (each addressing a specific pain they have)
- The #1 objection they'll have and how to address it
- CTA text that resonates with their role

Requirements:
- Use THEIR language, not yours (marketing managers say "pipeline" not "sales funnel")
- Lead with the outcome they're measured on
- Each persona's messaging should feel like it was written specifically for them
```

### Elevator Pitch Generator

```
Generate elevator pitches for [PRODUCT NAME] at different lengths.

Product: [WHAT IT DOES]
For: [WHO IT'S FOR]
Key benefit: [PRIMARY OUTCOME]
Differentiator: [WHAT MAKES IT UNIQUE]
Proof point: [TRACTION, METRIC, OR NOTABLE CUSTOMER]

Write:

1. One-liner (under 15 words):
   3 variations — for casual conversations, email signatures, social bios

2. 30-second pitch (50-75 words):
   3 variations — for networking events, cold intros, investor meetings

3. 60-second pitch (100-150 words):
   2 variations — for demo intros, conference talks, podcast appearances
   Include: Hook, problem, solution, proof, ask

4. Written bio (100 words):
   For press kits, conference programs, podcast descriptions

Requirements:
- No jargon in the one-liner (your grandparent should understand it)
- 30-second pitch should make someone say "tell me more"
- 60-second pitch should make someone say "can I try it?"
- Every version should include the benefit, not just what the product does
```

> **Why this works:** Most founders describe what their product does, not why anyone should care. "We use AI to analyze customer data" vs. "We help you predict which customers will churn before they do." Same product, completely different impact.

## Competitive Positioning Statements

### Positioning Statement Framework

```
Develop competitive positioning statements for [PRODUCT NAME].

Your positioning:
- Category: [WHAT CATEGORY YOU COMPETE IN]
- Target: [WHO YOU SERVE BEST]
- Differentiator: [YOUR UNIQUE ADVANTAGE]
- Proof: [EVIDENCE THAT SUPPORTS YOUR CLAIM]

Competitors:
- [COMPETITOR 1]: Known for [THEIR POSITIONING]
- [COMPETITOR 2]: Known for [THEIR POSITIONING]
- [COMPETITOR 3]: Known for [THEIR POSITIONING]

Generate:

1. Positioning statement:
   "For [TARGET] who need [JOB TO BE DONE], [PRODUCT NAME] is the only [CATEGORY] that [UNIQUE VALUE]. Unlike [COMPETITORS], [PRODUCT NAME] [PROOF POINT]."

2. Positioning by competitor (when a prospect mentions each competitor):
   - "[COMPETITOR 1]? They're great for [X]. We're built specifically for [Y], which means [BENEFIT]."
   - Repeat for each competitor

3. Category creation positioning (if applicable):
   "We don't think of ourselves as a [EXISTING CATEGORY]. We're building [NEW CATEGORY] — which means [REFRAME]."

4. Three positioning angles (for different contexts):
   - For sales calls: Lead with [ANGLE]
   - For marketing: Lead with [ANGLE]
   - For press/analysts: Lead with [ANGLE]

Requirements:
- Positioning should be honest and defensible
- Never position based on features alone — always tie to outcomes
- If a competitor is genuinely better at something, don't claim otherwise — own a different angle
```

### Sales Battle Cards

```
Create sales battle cards for [PRODUCT NAME] against key competitors.

Create a battle card for: [COMPETITOR NAME]

Sections:

1. Competitor Overview:
   - What they do (one sentence)
   - Their target market
   - Their pricing
   - Their funding/size (if relevant)

2. Where We Win:
   - [ADVANTAGE 1]: Specific scenario where we're clearly better + proof point
   - [ADVANTAGE 2]: Specific scenario + proof
   - [ADVANTAGE 3]: Specific scenario + proof

3. Where They Win:
   - [THEIR ADVANTAGE 1]: When they're genuinely better + how to handle it
   - [THEIR ADVANTAGE 2]: When they're better + how to handle it

4. Landmines (questions to ask prospects that highlight our strengths):
   - "How important is [CAPABILITY WE HAVE] to your team?"
   - "Have you run into issues with [THING COMPETITOR IS WEAK AT]?"
   - "What happens when you need [SCENARIO WHERE WE EXCEL]?"

5. Objection handling:
   - "But [COMPETITOR] is cheaper" — Response
   - "But [COMPETITOR] has [FEATURE]" — Response
   - "We've always used [COMPETITOR]" — Response
   - "Our team already knows [COMPETITOR]" — Response

6. Customer evidence:
   - "[CUSTOMER] switched from [COMPETITOR] to us because [REASON]"
   - Key metric from a switcher

Requirements:
- Keep each battle card to one page
- Focus on what the PROSPECT cares about, not what you wish they cared about
- Update quarterly as competitors evolve
```

## Upgrade & Upsell Messaging

### In-App Upgrade Prompts

```
Write in-app upgrade prompts for [PRODUCT NAME].

Write prompts for these trigger moments:

1. User hits a plan limit:
   - Soft wall: "You've reached [X] [THINGS] on the [PLAN] plan. Upgrade to [PLAN] for unlimited."
   - Hard wall: "You've used all [X] [THINGS] on your plan. Upgrade to continue."
   - For each: Headline + body + CTA button + secondary option

2. User tries a paid feature:
   - Teaser: Show a blurred preview of what they'd see
   - Copy: "This feature is available on [PLAN] and above."
   - CTA: "Upgrade to unlock" + "Learn more about plans"

3. User adds team members (approaching seat limit):
   - Banner: "[X] of [Y] seats used. Need more room?"
   - CTA: "Add seats" or "Upgrade plan"

4. Usage-based upsell:
   - "Your team processed [X] [UNITS] this month — that's [Y]% more than last month! Upgrade to [PLAN] for [HIGHER LIMIT]."

5. After a positive moment (completed a goal, received good analytics):
   - "Nice work! Want to do even more? [PLAN] includes [FEATURE THAT EXTENDS THIS SUCCESS]."

Requirements for all prompts:
- Never block or frustrate — guide and incentivize
- Always show what they GET, not what they're MISSING
- Include a dismiss option that doesn't feel punishing
- Max 30 words per prompt (headline + body)
```

### Upgrade Email Sequence

```
Write a 3-email upgrade sequence for [PRODUCT NAME] free plan users.

Context:
- Free plan includes: [WHAT'S FREE]
- Paid plan: [PLAN NAME] at $[PRICE]/mo
- Key paid features: [LIST 3-5]
- Typical trigger: [WHAT FREE USERS DO THAT INDICATES THEY NEED PAID]

Email 1 — Value education (Day 7 of free plan):
- Subject line: 3 options
- Show what's possible on the paid plan without hard selling
- Focus on one specific use case or outcome
- CTA: "See all [PLAN] features"
- 100-150 words

Email 2 — Social proof (Day 14):
- Subject line: 3 options
- Customer story or metric from a paid user similar to them
- "Here's what [SIMILAR COMPANY] achieved after upgrading"
- CTA: "Start your free trial of [PLAN]" (if applicable)
- 100-150 words

Email 3 — Limited offer (Day 21):
- Subject line: 3 options
- Time-limited incentive: [DISCOUNT, EXTENDED TRIAL, BONUS FEATURE]
- Create gentle urgency without being aggressive
- Final CTA with clear pricing and what's included
- 100-150 words
```

### Feature-Based Upsell Messages

```
Write upsell messages for [PRODUCT NAME] when users would benefit from a higher-tier feature.

For each scenario, write an in-app message and an email:

Scenario 1: User needs [FEATURE ON HIGHER PLAN]:
- Trigger: [WHAT THEY DID THAT INDICATES THEY NEED IT]
- In-app: Tooltip or modal (under 30 words)
- Email: 80-100 words explaining the feature and its value

Scenario 2: User's team is growing and needs [TEAM FEATURE]:
- Trigger: [INDICATOR — multiple logins, shared resources, collaboration attempts]
- In-app: Banner (under 20 words)
- Email: 80-100 words about team capabilities

Scenario 3: User needs [ADVANCED CAPABILITY]:
- Trigger: [WHAT THEY'RE TRYING TO DO]
- In-app: Feature teaser with "available on [PLAN]"
- Email: 80-100 words with a specific use case example

Requirements:
- Upsells should feel like genuine recommendations, not traps
- Time them to moments of need (not random Tuesdays)
- Always explain the VALUE of upgrading, not just the FEATURE
```

> **Pro tip:** The best upsell doesn't feel like a sales pitch — it feels like the product saying "I noticed you need more, and I can help." Behavior-triggered upsells convert 5-10x better than scheduled email blasts because they arrive at the moment of need.

## Annual vs Monthly Pitch Copy

### Pricing Toggle Copy

```
Write copy for the annual/monthly pricing toggle on [PRODUCT NAME]'s pricing page.

Annual discount: [PERCENTAGE]

Write:
1. Toggle labels: 3 options for how to label the monthly/annual switch
   - Example: "Monthly | Annual (Save 20%)"
   - Example: "Pay Monthly | Pay Yearly — 2 months free"
   - Example: "Monthly | Annual (Best value)"

2. Annual savings callout: A small badge or banner shown when annual is selected
   - 3 variations (under 10 words each)

3. Monthly-is-selected nudge: A subtle line shown when monthly is selected encouraging annual
   - 3 variations (under 15 words each)
   - Should be helpful, not guilting

4. Annual plan tooltip: Shown on hover over the annual option
   - What they save
   - What happens if they cancel
   - How billing works
   - Under 30 words
```

### Annual Conversion Email (For Monthly Subscribers)

```
Write an email sequence converting [PRODUCT NAME] monthly subscribers to annual.

Context:
- Monthly price: $[PRICE]/mo
- Annual price: $[PRICE]/mo billed annually ($[TOTAL]/year)
- Savings: $[AMOUNT]/year ([PERCENTAGE]%)
- Annual-only benefits: [LIST ANY EXTRAS, OR "SAME FEATURES"]
- Cancellation policy for annual: [WHAT HAPPENS IF THEY CANCEL MID-YEAR]

Email 1 — The math email:
- Subject line: 3 options
- Show the exact dollar savings over their remaining projected time as a customer
- "You've been with us for [X] months. If you stay another year (and we hope you will), switching to annual saves you $[AMOUNT]."
- Include a visual comparison table
- CTA: Switch to annual
- 120-160 words

Email 2 — The commitment email (1 week later):
- Subject line: 3 options
- Frame annual as a commitment to their OWN goals, not a billing preference
- "Annual subscribers are [X]% more likely to [ACHIEVE OUTCOME] because they commit and invest in the process"
- CTA: Lock in annual pricing
- 100-140 words

Email 3 — The deadline email (2 weeks later):
- Subject line: 3 options
- Create a time-limited incentive: "[X]% extra off if you switch by [DATE]"
- Simple, direct, final offer
- CTA: Claim the offer
- 80-120 words
```

### Annual Renewal Pitch

```
Write an annual renewal email sequence for [PRODUCT NAME].

Context:
- Renewal date: [DATE]
- Current annual price: $[PRICE]/year
- Renewal price: $[PRICE]/year (same / increased by [X]%)
- Usage metrics from the past year: [KEY METRICS]
- New features added during their year: [LIST 3-5]

Email 1 — 30 days before renewal:
- Subject line: 3 options
- Year in review: Highlight their usage metrics and value received
- Confirm renewal details: date, price, what's included
- If price increased: Explain the added value (new features, improvements)
- CTA: "Questions? Reply anytime" or "Update billing" link
- 150-200 words

Email 2 — 7 days before renewal:
- Subject line: 3 options
- Quick reminder of upcoming renewal
- Confirm payment method on file
- Reiterate one key benefit they get from staying
- CTA: "Review your subscription" link
- 80-120 words

Email 3 — Renewal confirmation:
- Subject line: "Your renewal is confirmed"
- Thank them for another year
- Preview what's coming in the next year
- CTA: Explore new features or share feedback
- 80-120 words
```

## Pricing Strategy Prompts

### Pricing Audit

```
Audit my current pricing strategy for [PRODUCT NAME].

Current pricing:
- [PLAN 1]: $[PRICE]/mo — [WHO IT'S FOR] — [FEATURES]
- [PLAN 2]: $[PRICE]/mo — [WHO IT'S FOR] — [FEATURES]
- [PLAN 3]: $[PRICE]/mo — [WHO IT'S FOR] — [FEATURES]

Metrics:
- Free-to-paid conversion: [%]
- Plan distribution: [% ON EACH PLAN]
- Monthly vs annual split: [%/%]
- Expansion revenue: [% OF TOTAL REVENUE FROM UPGRADES]
- Most common plan: [WHICH ONE]
- Churn rate by plan: [IF KNOWN]

Analyze:
1. Is my pricing aligned with the value I deliver? How can I tell?
2. Are my plans differentiated enough? Is there a "dead" plan nobody chooses?
3. Should I raise prices? By how much? Evidence-based reasoning.
4. Is my free plan too generous or too restrictive?
5. What pricing experiments should I run in the next quarter?
6. How does my pricing compare to [COMPETITOR 1] and [COMPETITOR 2]?
7. Should I add a plan, remove a plan, or restructure?
8. Is my annual discount too high or too low?
```

### Price Increase Communication

```
Write a price increase announcement for [PRODUCT NAME].

Context:
- Current price: $[OLD PRICE]/mo
- New price: $[NEW PRICE]/mo
- Increase: [PERCENTAGE]%
- Effective date: [DATE]
- Grandfathering: [YES — existing customers keep old price for [DURATION] / NO]
- Reason: [WHAT'S IMPROVED TO JUSTIFY THE INCREASE]
- Notice period: [HOW FAR IN ADVANCE — minimum 30 days]

Write:

1. Email to existing customers:
   - Subject line: 3 options (transparent, value-forward, straightforward)
   - Lead with the value delivered and improvements made
   - Announce the new pricing clearly
   - If grandfathered: Emphasize they're locked in at the current price for [DURATION]
   - If not grandfathered: Give ample notice and explain why
   - Offer to discuss or answer questions
   - 200-300 words
   - Tone: Confident and transparent — never apologetic about fair pricing

2. FAQ for the pricing change:
   - Why is the price going up?
   - When does the new pricing take effect?
   - Does this affect my current plan? (grandfathering details)
   - Can I lock in the old price by switching to annual?
   - What if I can't afford the new price?

3. Internal talking points: 5 bullet points for the support team to handle questions

Requirements:
- Give at minimum 30 days notice
- If you can grandfather existing customers (even temporarily), do it
- Never apologize for pricing that reflects value — but always acknowledge the change with empathy
```

> **Why this works:** Price increases are the highest-anxiety communication in SaaS. Done well (with notice, transparency, and value framing), most customers accept them without churning. Done poorly (surprise, no explanation), they trigger massive churn. The key is leading with value delivered, not cost imposed.

### Packaging Strategy Workshop

```
Help me redesign my pricing packaging for [PRODUCT NAME].

Current situation:
- Current plans: [LIST WITH PRICES AND FEATURES]
- What's working: [WHAT USERS LIKE ABOUT CURRENT PRICING]
- What's not working: [PAIN POINTS — confusing, too expensive, too many options, etc.]
- User segments: [WHO USES WHAT AND WHY]
- Goal: [WHAT I WANT THE NEW PACKAGING TO ACHIEVE]

Help me:
1. Identify the right number of plans (and why)
2. Define the "packaging axis" — what differentiates each plan (features, usage, seats, support level)
3. Determine which features should be:
   - Free (drives adoption)
   - Core (available on all paid plans)
   - Premium (worth paying more for)
   - Enterprise (custom pricing only)
4. Name each plan (avoid generic names — make them memorable or descriptive)
5. Set price points with reasoning (anchoring, value-based, competitive)
6. Design the upgrade path (what triggers users to upgrade naturally)
7. Identify any add-on opportunities (features priced separately)
8. Write the one-liner description for each plan
9. Predict potential issues with this packaging and how to mitigate
```
