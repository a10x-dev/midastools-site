## Set Your Store Context

```

Store: [STORE NAME]
Platform: [SHOPIFY / AMAZON / ETSY / WOOCOMMERCE / OTHER]
```

## Short-Form Descriptions

### Prompt 1: Marketplace Short Description (50 Words)

```
Write a 50-word product description for [PRODUCT NAME].

Product details:
- Category: [PRODUCT CATEGORY]
- Key feature: [SINGLE MOST IMPORTANT FEATURE]
- Material/composition: [WHAT IT'S MADE OF]
- Target buyer: [WHO THIS IS FOR]
- Price point: [PRICE]

Requirements:
- Exactly 50 words or fewer
- Lead with the strongest benefit
- Include one sensory detail (how it looks, feels, smells, etc.)
- End with a micro-CTA or urgency phrase
- No fluff words (amazing, wonderful, great, nice)
- Suitable for [AMAZON / ETSY / EBAY] marketplace listing
```

> **Why this works:** Marketplace shoppers scan fast. Leading with the benefit instead of the feature grabs attention. Sensory details create mental ownership — once a buyer imagines holding the product, they're halfway to purchasing.

### Prompt 2: Amazon Bullet Points

```
Write 5 Amazon-style bullet points for [PRODUCT NAME].

Product details:
- Category: [PRODUCT CATEGORY]
- Key features: [LIST 5-7 FEATURES]
- Materials: [MATERIALS/INGREDIENTS]
- Dimensions/size: [SIZE DETAILS]
- What's in the box: [PACKAGE CONTENTS]
- Target customer: [WHO BUYS THIS]
- Main competitor products: [1-2 COMPETING PRODUCTS]

Requirements:
- Start each bullet with a CAPITALIZED benefit phrase (2-4 words)
- Follow with a feature explanation
- Include specific numbers and measurements
- Address one common customer objection per bullet
- Keep each bullet to 200 characters max
- Naturally include these keywords: [KEYWORD 1], [KEYWORD 2], [KEYWORD 3]
```

> **Pro tip:** Amazon's algorithm weights bullet points heavily. Front-load each bullet with a keyword-rich benefit phrase. Think of each bullet as a mini-ad answering "why should I buy this?"

## Long-Form Product Page Copy

### Prompt 5: Full Product Page Description

```
Write a complete product page description for [PRODUCT NAME] on my [PLATFORM] store.

Product details:
- Category: [PRODUCT CATEGORY]
- Price: [PRICE]
- Key features: [LIST 4-6 FEATURES]
- Materials/ingredients: [COMPOSITION]
- Dimensions/size options: [SIZES AVAILABLE]
- Colors/variants: [OPTIONS]
- Target customer: [IDEAL BUYER PERSONA]
- Problem it solves: [PAIN POINT]
- What makes it different: [UNIQUE SELLING PROPOSITION]

Write in this structure:
1. **Headline** — benefit-driven, 8-12 words
2. **Opening hook** — 2-3 sentences that speak to the customer's pain point or desire
3. **Key benefits section** — 4-6 bullet points, each leading with a benefit followed by the feature
4. **Product story** — 3-4 sentences about why this product exists and what went into creating it
5. **Social proof placeholder** — "[X] customers love this" or "Rated [X] stars" with a brief quote
6. **Specifications** — clean list of technical details
7. **FAQ section** — 3 common questions with concise answers
8. **CTA** — a compelling reason to buy now

Brand voice: [BRAND VOICE]
Total length: 300-400 words
```

> **Why this works:** This structure mirrors how customers actually read product pages — they scan the headline, check benefits, look for proof, then read details only if interested. Matching their reading pattern means they hit a buying trigger at every scroll point.

```
Write a story-driven product description for [PRODUCT NAME].

The story angle: [CHOOSE ONE]
- Origin story (how this product came to be)
- Customer transformation (before/after using it)
- Day-in-the-life (showing the product in use)

Product details:
- What it is: [PRODUCT DESCRIPTION]

Requirements:

- Open with a scene or moment the reader can picture
- Weave product features into the story naturally (never list them)

- Brand voice: [BRAND VOICE]
```

### Prompt 7: Comparison-Driven Description

```
Write a product description for [PRODUCT NAME] that positions it against alternatives without naming competitors.

My product advantages:
- Feature 1: [MY FEATURE] vs typical [INFERIOR ALTERNATIVE]
- Feature 2: [MY FEATURE] vs typical [INFERIOR ALTERNATIVE]
- Feature 3: [MY FEATURE] vs typical [INFERIOR ALTERNATIVE]

Product details:
- Price: [PRICE]
- Target buyer: [CUSTOMER WHO IS COMPARISON SHOPPING]
- Main objection: [WHY THEY MIGHT NOT BUY]

Requirements:
- 150-250 words
- Use "unlike most [PRODUCT CATEGORY]" framing, never name competitors
- Highlight differences as benefits, not just features
- Include a "you deserve better" undertone without being arrogant
- Add a money-back guarantee or trust signal
- Close with urgency or scarcity if appropriate
```

> **Pro tip:** Comparison copy works best for products in crowded markets where buyers are actively shopping between options. It intercepts the comparison mindset and does the work for them.

## SEO-Optimized Descriptions

### Prompt 9: SEO Product Description

```
Write an SEO-optimized product description for [PRODUCT NAME].

SEO details:
- Primary keyword: [MAIN KEYWORD — e.g., "organic cotton baby blanket"]
- Secondary keywords: [KEYWORD 2], [KEYWORD 3], [KEYWORD 4]
- Search intent: [WHAT THE SEARCHER IS LOOKING FOR]
- Current ranking competitors: [TOP 3 COMPETITOR PRODUCTS FOR THIS KEYWORD]

Product details:
- Category: [CATEGORY]
- Key features: [3-5 FEATURES]
- Price: [PRICE]
- Unique angle: [WHAT MAKES YOURS DIFFERENT]

Requirements:
- 250-350 words
- Include primary keyword in the first sentence and 2-3 more times naturally
- Include each secondary keyword at least once
- Write for humans first, search engines second
- Use H2 and H3 subheadings that include keywords
- Include a question-based subheading (matches "People also ask" format)
- Write a meta description (155 characters) separately at the end
- Suggest an SEO-friendly URL slug
```

> **Why this works:** Google's algorithms prioritize content that matches search intent. By structuring your description around what buyers actually search for, you rank higher AND convert better because the copy directly addresses their needs.

## Luxury / Premium Tone Variants

### Prompt 11: Premium Brand Description

```
Write a luxury-tier product description for [PRODUCT NAME].

Product details:
- Category: [PRODUCT CATEGORY]
- Materials: [PREMIUM MATERIALS — e.g., Italian leather, Japanese steel, Swiss movement]
- Craftsmanship detail: [HANDMADE / LIMITED EDITION / ARTISAN-CRAFTED]
- Price: [PRICE]
- Heritage/origin: [WHERE IT'S MADE OR BRAND STORY]
- Exclusivity factor: [LIMITED QUANTITY / MADE TO ORDER / SEASONAL]

Requirements:
- 200-300 words
- Evoke aspiration and exclusivity
- Use sensory language (the weight, the texture, the sound)
- Reference craftsmanship and time (never rush)
- Avoid hard-sell tactics — let the product speak
- No exclamation marks, no ALL CAPS, no urgency language
- Short, elegant sentences
- Suggest it's an investment, not a purchase
- Make the reader feel they're joining an exclusive group
- Brand voice: refined, understated, confident
```

> **Pro tip:** Luxury copy sells the experience and identity, never the product. A $400 candle description should make you feel like the kind of person who owns a $400 candle — aspirational identity drives premium purchases.

## Budget / Value Tone Variants

### Prompt 14: Value-Driven Description

```
Write a value-focused product description for [PRODUCT NAME].

Product details:
- Category: [CATEGORY]
- Price: [PRICE]
- Comparable products cost: [COMPETITOR PRICE RANGE]
- Key features: [FEATURES THAT MATCH HIGHER-PRICED ALTERNATIVES]
- Why it's affordable: [DIRECT TO CONSUMER / NO MIDDLEMAN / EFFICIENT MANUFACTURING]
- Quality assurance: [WARRANTY / GUARANTEE / RETURN POLICY]

Requirements:
- 150-200 words
- Lead with the value proposition (what they get for the price)
- Use "smart buyer" language — make them feel savvy, not cheap
- Compare value, never just price (cost per use, longevity, etc.)
- Include a trust-builder (guarantee, reviews, return policy)
- Address the "too good to be true" objection directly
- Tone: confident, no-nonsense, straight talk
```

> **Why this works:** Budget buyers have different objections than premium buyers. They worry about quality, not exclusivity. Addressing the "catch" directly and framing the purchase as smart builds the trust needed to convert.

```
Write a tech product description for [PRODUCT NAME].

Product details:
- Category: [GADGET / ACCESSORY / COMPONENT / SOFTWARE]
- Key specs: [TOP 5 SPECIFICATIONS]
- Compatibility: [WHAT IT WORKS WITH]
- Battery life / power: [BATTERY DETAILS]
```

> **Pro tip:** Tech buyers research heavily. Your description needs to answer every spec question while still telling an emotional story about what the product enables. Specs decide, benefits persuade.

## A/B Testing Variants

### Prompt 23: A/B Test — Benefit vs Feature Lead

```
Write 2 versions of a product description for [PRODUCT NAME] to A/B test.

Product details:
- Key features: [LIST 3-5 FEATURES]
- Key benefits: [LIST 3-5 BENEFITS]
- Target customer: [WHO BUYS THIS]
- Price: [PRICE]

Version A — Benefit-Led:
- Open with the biggest transformation or benefit
- Features support the benefit claims
- Emotional, aspirational language
- 150-200 words

Version B — Feature-Led:
- Open with the most impressive feature or spec
- Benefits explain why each feature matters
- Logical, credible language
- 150-200 words

Make both versions equally compelling but clearly different in approach. Label which type of buyer each version is likely to convert better.
```

> **Pro tip:** Test benefit-led copy against feature-led copy. Generally, benefit-led wins for lifestyle products while feature-led wins for technical products — but let your data confirm. Run each version for at least 500 visitors before declaring a winner.

```
Product details:

- Full story + benefits + specs + social proof + FAQ
- For buyers who need convincing
- Multiple sections with headers
- For higher-priced or complex products

Explain when to use each: which product types, price points, and customer types favor short vs long descriptions.
```

### Prompt 27: Variant Descriptions

```
Write unique descriptions for each variant of [PRODUCT NAME].

Base product: [PRODUCT DESCRIPTION]
Variants:
1. [COLOR/SIZE/STYLE 1]
2. [COLOR/SIZE/STYLE 2]
3. [COLOR/SIZE/STYLE 3]
4. [COLOR/SIZE/STYLE 4]

For each variant, write:
- A 2-sentence variant-specific description that highlights what makes this option unique
- A suggested "best for" tag (e.g., "Best for date night," "Best for everyday wear")
- One styling or use suggestion specific to this variant

Don't just swap the color name — give each variant its own personality while maintaining the base product's core message.
```

> **Why this works:** Most stores use identical descriptions across variants, just swapping the color name. Unique variant descriptions reduce bounce rate and help customers choose, which reduces returns and increases conversion.
